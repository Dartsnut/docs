# Submit For Review

Use the Community panel to submit the active game or widget workspace for
review. Submit only from a saved or opened workspace that contains the final
files you want reviewers to evaluate.

## Before You Submit

1. Save the project to a permanent folder if it is still temporary.
2. Run it in the emulator.
3. Deploy to a machine if the behavior depends on real hardware.
4. Capture or prepare at least one preview image.
5. Make sure `conf.json` describes the correct game or widget.

## Basic Flow

1. Open the workspace you want to submit.
2. Open the **Community** panel.
3. Sign in if the app asks you to.
4. Click **Refresh** if the panel has not loaded the current workspace details.
5. Review or fill in the app name and app ID.
6. Choose a category.
7. Choose the control type. For widgets, choose all controls that apply.
8. For widgets, confirm the widget size.
9. Set the version.
10. Add review notes.
11. Upload an icon.
12. Add one or more preview images.
13. Click **Submit Game Version for Review** or **Submit Widget Version for
    Review**.

After submission, the version appears under **My Submissions** with its review
status.

## Required Information

| Field | Notes |
| --- | --- |
| App name | The display name shown to users. |
| App ID | The stable identifier for this game or widget. |
| Category | Where the submission should appear. |
| Control | How users interact with the game or widget. |
| Widget size | Required for widgets. |
| Version | The version being reviewed, such as `1.0.0`. |
| Review notes | Short notes for reviewers about what changed or what to check. |
| Icon | Required cover image. |
| Preview images | Required screenshots or previews. |

## Existing Submissions

The panel shows existing versions for the current app ID. If a version is
already in review, submit a new version only after pulling the current one out
of review or waiting for the review to finish.

Use **Pull out of review** when:

1. You submitted the wrong files.
2. You need to fix a problem before reviewers continue.
3. You want to replace the in-review version with a newer one.

## When To Submit

Submit for review when:

1. The project runs correctly in the emulator.
2. Hardware-only behavior has been checked on a machine when needed.
3. The icon and preview images represent the current version.
4. The review notes explain what reviewers should look at.
5. You are ready for the version to enter the community review process.
